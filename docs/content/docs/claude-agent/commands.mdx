# Commands Reference

The Claude Agent provides two slash commands that wrap the five-phase workflow. Both commands use natural language intent routing -- you describe what you want and the agent maps your request to the appropriate scripts.

## /linkedin-prospector

The Pull agent. Handles configuration, searching, enrichment, and database management (Phases 0 and 1).

### Intent Routing Table

| What to Say | Intent | What Happens |
|-------------|--------|--------------|
| `set up my ICP config` | configure | Starts 8-step conversational ICP configuration |
| `validate config` | validate | Checks config file validity |
| `find me 20 [niche] contacts` | search | Searches LinkedIn 1st-degree connections by niche |
| `search for [role/industry/keyword]` | search | Targeted LinkedIn search with custom keywords |
| `enrich unenriched contacts` | enrich | Visits profiles to pull detailed data |
| `how many contacts do I have?` | stats | Shows database statistics |
| `export my contacts as CSV` | export | Exports contacts in CSV or JSON format |
| `rebuild and score everything` | pipeline | Delegates to `pipeline.mjs --rebuild` |
| `reparse cached pages` | reparse | Re-extracts data from cached HTML |
| `log me into LinkedIn` | login | Launches browser for manual LinkedIn login |
| `search for "natural language"` | db-search | Queries contacts with vector-enhanced search |
| `vectorize my contacts` | vectorize | Builds the RVF vector store |

### How the Agent Works

1. **Checks the local DB first** -- Runs `db.mjs stats` and `db.mjs search` to see if matching contacts already exist.
2. **Searches only when needed** -- If the DB lacks enough contacts, runs `search.mjs` with the appropriate niche or keywords.
3. **Enriches profiles** -- Visits each un-enriched profile to extract headline, about, current role/company, and connection count.
4. **Caches everything** -- All LinkedIn pages are cached as raw HTML in `data/cache/` for offline re-extraction.

### Search Syntax

By niche (using keywords from `icp-config.json`):
```
/linkedin-prospector find me 20 ecommerce contacts
/linkedin-prospector search for SaaS founders
```

By custom keywords:
```
/linkedin-prospector search for "machine learning engineer" contacts
/linkedin-prospector find CIOs at healthcare companies who mention Epic
```

### Example Conversations

**Setting up configuration:**
```
User: /linkedin-prospector set up my ICP config
Agent: What services do you offer?
User: vCISO services and managed security assessments
Agent: For "vCISO services", who are your target buyers?
User: CISO, CTO, VP Engineering, IT Director
Agent: What industries are your ideal customers in?
User: fintech, healthtech, SaaS startups
Agent: What keywords signal they need your services?
User: SOC 2, compliance, security audit, cyber insurance
Agent: Config written. 2 profiles, 3 niches. Validation passed.
       Run `/linkedin-prospector find me startup-cto contacts` to start.
```

**Searching and enriching:**
```
User: /linkedin-prospector find me 15 health-it-leaders contacts
Agent: Checked DB -- 2 existing contacts. Searching LinkedIn...
       Found 16 new profiles. Total: 18 health IT contacts.
       Would you like me to enrich them?

User: /linkedin-prospector enrich unenriched contacts
Agent: Enriching 16 profiles... Done. 16/16 enriched.
```

## /network-intel

The Analyze/Report agent. Handles scoring, analysis, expansion, and reporting (Phases 2-4).

### Intent Routing Table

| What to Say | Intent | What Happens |
|-------------|--------|--------------|
| `rebuild and rescore everything` | bootstrap | Full graph build + all 3 scoring layers |
| `rescore everything` | score | Rescore without rebuilding graph |
| `who are my best hubs?` | hubs | Top contacts by network influence |
| `find my top prospects` | prospects | Top contacts by ICP fit |
| `who are my best referral partners?` | referrals | Referral partner analysis |
| `what should I focus on next?` | recommend | Strategic recommendations |
| `give me an overview` | summary | Network summary with distributions |
| `show me super-connectors` | behavioral | Behavioral analysis with persona filter |
| `content visibility strategy` | visibility | Content strategy recommendations |
| `which companies have the best network value?` | employers | Company ranking by ENV score |
| `show me clusters` | clusters | Industry/niche cluster mapping |
| `show me contacts at [Company]` | company | Company deep-dive |
| `find contacts similar to [URL]` | similar | k-NN vector similarity search |
| `who talks about [topic]?` | semantic | Semantic free-text search |
| `deep scan [URL]` | deep-scan | Discover a contact's 2nd-degree connections |
| `expand network through referral partners` | expand | Batch deep-scan by criteria |
| `any new connections since last time?` | delta | Change detection vs. last snapshot |
| `generate a report` | report | Interactive HTML dashboard |
| `export my contacts` | export | JSON/CSV export |
| `start dashboard` | dashboard | Launch the local Next.js dashboard app |
| `stop dashboard` | dashboard-stop | Stop the running dashboard |
| `vectorize my contacts` | vectorize | Build the RVF vector store |

### Dashboard Management

The agent can start, stop, and query a local Next.js dashboard at port 3100:

```
/network-intel start dashboard
/network-intel is dashboard running?
/network-intel stop dashboard
```

The dashboard exposes REST APIs for querying data:

| Endpoint | Purpose |
|----------|---------|
| `GET /api/dashboard` | KPIs, top contacts, suggested actions |
| `GET /api/contacts?tier=gold&sort=goldScore` | Query contacts with filters |
| `GET /api/contacts/[slug]/similar` | Find similar contacts |
| `GET /api/search?q=keyword` | Search contacts |
| `GET /api/graph` | Network graph data for visualization |
| `POST /api/actions/run` | Trigger scripts via the dashboard |
