# Claude Agent

The Claude Agent is a portable [Claude Code](https://docs.anthropic.com/en/docs/claude-code) skill that provides LinkedIn network intelligence through two slash commands. It searches your 1st-degree connections, enriches profiles, builds a knowledge graph, scores contacts across three layers, and generates actionable analysis -- all from conversation.

## Three Subsystems

NetworkNav is built from three cooperating subsystems, each with a distinct role:

```
+------------------+        +------------------+        +------------------+
|   Web App        |        |   Claude Agent   |        | Browser Extension|
|   (app/)         |<------>|   (agent/)       |<------>|   (browser/)     |
|                  |        |                  |        |                  |
| Next.js dashboard|        | Claude Code skill|        | Chrome MV3       |
| PostgreSQL + RVF |        | Playwright + CLI |        | Page capture     |
| Scoring UI       |        | Scoring engine   |        | Task display     |
| Outreach mgmt    |        | 12 analysis modes|        | Outreach overlay |
| REST API         |        | Semantic search  |        | Sidebar panel    |
+------------------+        +------------------+        +------------------+
```

| Subsystem | Role | Data Flow |
|-----------|------|-----------|
| **Web App** | Web UI and persistent storage. Dashboard, contacts table, scoring visualization, enrichment management, outreach pipeline, REST API. | Stores all contacts in PostgreSQL. Serves the UI. Receives captured data from the extension. |
| **Claude Agent** | CLI intelligence layer. Active LinkedIn search via Playwright, 3-layer scoring engine, 12 analysis modes, semantic vector search, ICP configuration, report generation. | Reads/writes local JSON files. Pushes scored data to the app via API. Operates independently for offline analysis. |
| **Browser Extension** | Passive data capture. Extracts profile data from LinkedIn pages you visit, displays tasks, provides outreach templates inline. | Sends captured page data to the app API. Displays tasks and outreach context from the app. |

The agent operates independently of the app and extension -- it maintains its own local data store and can run entirely offline. When the app is available, scored contacts and analysis results sync to PostgreSQL for visualization in the dashboard.

## Five-Phase Workflow

```
Phase 0: CONFIGURE  -- Define ICP profiles, scoring weights, niche definitions
Phase 1: PULL       -- Search LinkedIn, enrich profiles, cache locally
Phase 2: SCORE      -- Build knowledge graph, run 3-layer scoring engine, vectorize
Phase 3: ANALYZE    -- 12 analysis modes (hubs, prospects, referrals, semantic...)
Phase 4: REPORT     -- Interactive HTML dashboard with 3D graph and charts
```

Two slash commands wrap these phases:

| Command | Phases | Purpose |
|---------|--------|---------|
| `/linkedin-prospector` | 0 and 1 | Configure ICP profiles, search LinkedIn, enrich profiles, manage the contact database |
| `/network-intel` | 2, 3, and 4 | Build the graph, score contacts, run analysis, generate reports, expand the network |

## Three-Layer Scoring Engine

The agent scores every contact across three independent layers that build on each other sequentially:

| Layer | What It Measures | Key Outputs |
|-------|-----------------|-------------|
| **Layer 1: ICP + Gold Score** | Role fit, industry match, buying signals, network centrality, relationship strength | Gold Score, Tier (gold/silver/bronze/watch), ICP Persona (buyer/advisor/hub/peer) |
| **Layer 2: Behavioral** | Connection power, recency, about-section signals, headline patterns, super-connector traits, amplification potential | Behavioral Score, Behavioral Persona (super-connector/content-creator/silent-influencer/rising-connector) |
| **Layer 3: Referral** | Referral role, client overlap, network reach, amplification power, relationship warmth, buyer inversion | Referral Likelihood, Referral Tier, Referral Persona (warm-introducer/white-label-partner/co-seller/amplifier) |

## Semantic Search with ruvector

The agent integrates [ruvector](https://github.com/ruvnet/ruvector) as a core dependency for semantic vector search. Every contact's profile text is converted into a 384-dimensional embedding using the all-MiniLM-L6-v2 ONNX model and stored in an HNSW-indexed vector store. This enables finding contacts by meaning rather than exact keyword matches.

See [Semantic Search](/docs/claude-agent/semantic-search) for details.
